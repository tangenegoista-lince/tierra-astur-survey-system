# Report Finisher

Take messy or intermediate work and turn it into a strong final deliverable.

This skill is for the last mile:
- cleaning structure
- tightening language
- upgrading weak notes into a real document
- choosing the right output format
- making conclusions usable

## Core workflow

1. **Identify the output type**
   Decide what the user actually needs:
   - executive summary
   - technical report
   - applied recommendation
   - decision brief
   - action plan
   - concise summary for messaging

2. **Identify the source material quality**
   Determine whether the input is:
   - raw notes
   - partial draft
   - solid analysis with poor presentation
   - good findings needing a shorter version
   - mixed material needing synthesis

3. **Restructure before rewriting**
   Before polishing sentences, fix:
   - ordering
   - heading logic
   - duplication
   - missing transitions
   - weak conclusion sections

4. **Choose the right level of compression**
   Decide whether to:
   - preserve depth
   - produce an executive version
   - create both short and long versions

5. **Sharpen judgment**
   Upgrade the output by making sure it includes:
   - what matters most
   - what is evidence vs interpretation
   - what to do next
   - what is uncertain

6. **Make the document reusable**
   For substantial work, write a file in the workspace.
   Use the relevant template from `templates/` when useful.

## Editing principles

- Cut repetition aggressively.
- Prefer strong section titles over long transitions.
- Replace vague conclusions with explicit judgment.
- If the text is technically correct but hard to use, improve usability first.
- Do not inflate the document with filler.
- Preserve nuance, but do not hide the answer.

## Executive-output rules

When producing an executive version:
- start with the answer
- keep the number of main points small
- focus on implications and decisions
- avoid drowning the reader in source-by-source detail

## Technical-output rules

When producing a technical version:
- preserve important caveats
- keep methodology visible when it matters
- separate facts, inferences, and recommendations
- include enough structure that the document can be extended later

## Applied recommendation rules

When the task is strategy or adoption oriented, make sure the deliverable includes:
- what to copy
- what to adapt
- what to ignore
- what to build first
- why

## Good Report Finisher outcomes

A good finished report should:
- read like a final deliverable, not a working note
- save the user time
- support a decision or next action
- be easy to reuse later
