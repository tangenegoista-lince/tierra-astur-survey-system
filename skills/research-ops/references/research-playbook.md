# Research Playbook

## Default sequence

1. Restate the research question in one sentence.
2. Decide whether local context matters.
3. Gather official sources first.
4. Add accredited technical sources.
5. Add curated community sources for breadth.
6. Compare claims and note contradictions.
7. Extract repeated patterns.
8. Write the report.
9. Finish with recommendations.

## Good questions to answer while researching

- What is official vs unofficial here?
- What workflows or use cases repeat across sources?
- What features appear mature rather than aspirational?
- Where is there obvious hype inflation?
- What matters specifically for Lince/OpenClaw?

## For GitHub/repo research

Check:
- owner identity
- whether the repo is official
- stated purpose
- README claims
- docs linked from the repo
- ecosystem references around it
- whether it looks canonical, derivative, experimental, or marketing-heavy

## For strategy outputs

Always convert findings into decisions:
- copy
- adapt
- ignore
- build next
